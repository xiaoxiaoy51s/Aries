# TRAE Security Review - 安全审查技能

## 适用场景

审查代码中的安全漏洞、敏感数据泄露、注入攻击、认证授权缺陷、配置安全等问题。

## 安全审查清单

### 1. 注入攻击

- SQL 注入：是否使用了参数化查询？
- 命令注入：是否对 shell 命令参数做了转义？
- OS 注入：是否验证了文件路径？
- LDAP / NoSQL 注入：输入是否正确过滤？

### 2. 认证与授权

- 是否有硬编码密码或密钥？
- 会话管理是否安全？Token 是否合理过期？
- 权限校验是否在每个接口都执行？
- 是否存在越权漏洞？

### 3. 敏感数据泄露

- 日志中是否打印了密码、Token、密钥？
- 敏感信息是否在 URL 参数中传输？
- 错误信息是否暴露了内部实现细节？
- API 响应是否返回了不必要的敏感字段？

### 4. 输入验证

- 用户输入是否做了长度、格式、范围校验？
- 文件上传是否有类型和大小限制？
- 是否有 SSRF（服务端请求伪造）防护？

### 5. 配置安全

- 是否使用了默认配置或默认密码？
- CORS 配置是否过于宽松？
- HTTPS 是否强制启用？
- 依赖是否有已知安全漏洞？

## 输出格式

```markdown
## 安全审查报告

### 风险等级分布
- Critical: N 个
- High: N 个
- Medium: N 个
- Low: N 个

### 详细发现

#### [等级] 问题标题
- **位置**: 文件:行号
- **描述**: 问题说明
- **风险**: 潜在影响
- **建议**: 修复方案
- **参考**: [CWE/OWASP 编号]

### 安全评分
[优/良/中/差]
```
