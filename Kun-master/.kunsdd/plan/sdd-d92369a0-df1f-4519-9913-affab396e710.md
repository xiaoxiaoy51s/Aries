# Plan: 将 Code/Write 模式按钮从上下排改为左右等宽排列

## 概要

将 `WorkspaceModeTabs` 组件中的 **Code** 和 **Write** 两个模式切换按钮从当前的**上下垂直排列**改为**左右水平排列**，并让两个按钮**等宽**。

## 影响范围

仅涉及一个组件文件，无 API 变更，无 prop 变更，消费方无需修改：

| 文件 | 变更类型 |
|------|----------|
| `src/renderer/src/components/chat/WorkspaceModeTabs.tsx` | 修改布局 CSS |
| `src/renderer/src/components/chat/__tests__/WorkspaceModeTabs.test.tsx` | 新增测试 |

消费方（无需修改）：
- `src/renderer/src/components/chat/Sidebar.tsx` (line 131)
- `src/renderer/src/components/write/WriteSidebar.tsx` (line 253)

## 实现步骤

### Step 1: 修改容器布局方向

**文件**: `src/renderer/src/components/chat/WorkspaceModeTabs.tsx`

将容器 div 的 `flex-col` 改为 `flex-row`（或直接用 `flex`，默认即为 row）：

```diff
- className="mb-1 flex flex-col gap-0.5"
+ className="mb-1 flex flex-row gap-0.5"
```

### Step 2: 让两个按钮等宽

当前每个按钮有 `w-full`，在垂直布局下各自撑满宽度。改为水平布局后，`w-full` 会导致每个按钮都尝试占满整行，造成溢出。需要改为 `flex-1` 让两个按钮均分可用宽度：

在 `tabClass` 函数中（或直接在按钮 className 中），将 `w-full` 替换为 `flex-1 min-w-0`：

```diff
- `inline-flex min-h-[32px] w-full min-w-0 items-center gap-2 ...`
+ `inline-flex min-h-[32px] flex-1 min-w-0 items-center gap-2 ...`
```

`min-w-0` 保留以确保文本截断 (`truncate`) 在窄容器中正常工作。

### Step 3: 添加单元测试

**文件**: `src/renderer/src/components/chat/__tests__/WorkspaceModeTabs.test.tsx`（新建）

测试用例：
1. **渲染两个按钮** — 验证 Code 和 Write 按钮都渲染
2. **水平布局** — 验证容器使用 `flex-row`（或 `flex` 默认 row），而非 `flex-col`
3. **等宽** — 验证两个按钮都有 `flex-1` class，确保均分宽度
4. **active 状态** — 验证 `aria-selected` 正确反映 `activeView` prop
5. **点击回调** — 验证点击 Code 按钮触发 `onCodeOpen`，点击 Write 按钮触发 `onWriteOpen`

## 风险

| 风险 | 等级 | 缓解 |
|------|------|------|
| 侧边栏宽度较窄时按钮文字可能显示不全 | 低 | `min-w-0` + `truncate` 已处理；图标 `shrink-0` 保证不被压缩 |
| 两个按钮文字长度不同（Code vs 写作）导致视觉不均 | 低 | `flex-1` 确保等宽，中文"写作"稍长但仍在按钮宽度内 |

## 不做

- 不修改 prop 接口
- 不修改消费方 Sidebar / WriteSidebar
- 不修改 i18n 翻译
- 不改变按钮的交互行为（点击、aria、role）
