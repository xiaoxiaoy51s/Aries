# DeepSeek GUI v0.2.1

这一版是 v0.2.0 之后的一次稳定性与交付体验补强。重点不在新增大功能，而是在把 Kun 运行时、微信桥接和打包发布链路做得更稳、更自包含，让日常使用和升级更少踩坑。

### 更稳的 Kun 启动与配置迁移

- 启动时会等待明确的 `KUN_READY` 就绪标记，避免“进程已起但服务还没真正可用”的假启动。
- GUI 管理的 Kun 配置会先清理历史脏字段，再按 schema 校验后写入，历史损坏配置不再容易把运行时带偏。
- 旧版 `agentProvider`、`deepseek`、`codewhale`、`reasonix` 等设置会自动迁移到新的 `agents.kun` 结构。
- 旧的本地 HTTP 默认端口 `7878` 会自动升级到 Kun 默认端口 `8899`，减少升级后的端口冲突和误连。
- 已经手动关闭的 attachments、web、MCP 等能力开关会继续保留，不会被 GUI 强行改回去。

### 微信连接链路重写

- 微信桥接改为内置 HTTP bridge，发布包不再依赖外部 OpenClaw 适配层或额外的 Node 二进制。
- 内置了 `@tencent-weixin/openclaw-weixin` 的兼容 shim，微信登录、拉取消息和回包链路都直接由 GUI 管理。
- 登录流程支持二维码生成、轮询确认、账号状态持久化和多账号恢复。
- 收消息后会保留 context token，并通过 GUI webhook 回传，让后续对话更连续。
- 旧的微信状态文件和 token 会尽量兼容迁移，升级后通常只需要重新连一次微信即可。

### 打包与发布更可靠

- 发布包现在会把 Kun runtime 的 `dist`、`package.json`、`package-lock.json` 和 `node_modules` 一起打进去，减少缺依赖问题。
- after-pack 阶段会在交付前验证 Kun 运行时是否齐全，并清理多余的开发依赖。
- macOS 和 Windows 的发布流程已经拆开，产物生成与上传更清晰，也更不容易混包。
- 这版发布包对 `openclaw` 兼容 shim、Kun runtime 和 SQLite 依赖的处理更加明确，整体自包含程度更高。

### 升级说明

- 如果你从旧版本升级，首次启动时会自动迁移大部分本地设置。
- 如果你之前使用微信接入，建议在升级后重新连一次微信，以便刷新新的桥接状态。
- 如果你维护自己的 Kun 配置，`0.2.1` 之后建议尽量让 GUI 接管相关配置写入，不要再手工拼接旧版字段。

### 总结

v0.2.1 是一次偏“把底座磨平”的版本。它没有改变 DeepSeek GUI 的主产品方向，但把 Kun 的启动、配置迁移、微信桥接和发布交付都做得更稳，适合从 `0.2.0` 平滑升级到更可用的日常版本。
